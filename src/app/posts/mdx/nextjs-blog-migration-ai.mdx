처음 블로그를 시작한 건 **2021년**이었습니다. 약 1년간 운영하다가 중단했지만, 이제 다시 새롭게 시작하려고 합니다.  
그때도 지금과 비슷한 마음으로 출발했지만, 솔직히 말해 당시엔 <strong>“개발자는 블로그를 해야 한다”</strong>는 막연한 의무감이 더 컸습니다.  

이번에는 달라지려고 합니다. **스스로의 성장과 학습 루틴을 다시 세우기 위해**, 그리고 경험을 체계적으로 남기기 위해 블로그를 새롭게 정비했습니다. 
앞으로는 결과보다 과정을, 기록보다 **성장**을 더 중시하는 블로그를 지향합니다.

---

### 🚀 Next.js로 이전한 이유
기존 정적 블로그는 유지보수가 쉽지만, 커스터마이징에 한계가 있었습니다.  
Next.js는 다음과 같은 장점이 있었죠.

- React 기반의 유연한 UI 구성  
- MDX 지원으로 **코드와 문서를 함께 관리 가능**  
- GitHub Actions를 이용한 자동 배포  
- 다크모드, 카테고리, 검색 등 기능 확장 용이  

덕분에 “내가 직접 만든 나만의 기술 블로그”라는 느낌이 강해졌습니다.

---

### 🤖 AI 툴이 도와준 부분
이번 이전에서 가장 인상 깊었던 점은 <strong>AI 도구(ChatGPT, Gemini 등)</strong>을 활용하면서  
“생각보다 오랜 시간이 걸리지 않았다”는 것이었습니다.  

Windows 개발자로서 웹 분야에 대한 깊은 경험이 부족해 개발 기간이 오래 걸릴 것이라 예상했지만,  
AI의 도움 덕분에 단 **3일 만에 만족스러운 결과물을 완성**할 수 있었습니다.   

- GitHub Actions 설정 오류 → AI가 바로 YAML 수정 예시 제공  
- Tailwind CSS 다크모드 전환 버그 → AI가 원인 분석 및 코드 수정 제안  
- 페이지 라우팅 구조 설계 → 단계별 코드 템플릿 제안  

덕분에 막히는 구간마다 “검색 + 시행착오” 대신 “이해 + 실행”으로 해결할 수 있었습니다.  
개발자는 **모든 걸 혼자 해결할 필요는 없고, 잘 질문하는 능력이 더 중요하다**는 걸 다시 느꼈습니다.

---

### 💡 느낀 점
이번 이전을 통해 **AI를 곁에 둔 개발의 시대**가 이미 현실임을 체감했습니다.  
Next.js의 구조를 완전히 이해하지 못하더라도, AI와 협업하면 빠르게 프로토타입을 만들 수 있습니다.  
무엇보다, 내가 만든 블로그를 통해 다시 배우고 공유할 수 있다는 점이 가장 큰 보람이었습니다.

---

### 🌱 앞으로의 계획
이 블로그는 이제 단순한 글 모음이 아니라 제가 걸어온 길을 기록하고, 또 앞으로의 성장 과정을 보여주는 공간이 될 것입니다. 
앞으로도 꾸준히, 그리고 꾸밈없이 기록하겠습니다.
